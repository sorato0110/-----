# デバイス（ブラウザ）に残るデータの救出・復旧計画

現在、App Engineサーバー (`haze-strategy-web.an.r.appspot.com`) が課金エラーにより「503 サーバーエラー」で開けない状態になっています。
ユーザーのこれまでのデータは、このURLに紐づく形で **各デバイス（スマホやPC）のブラウザ内部（LocalStorage）** に保存されています。
サーバーが開けないと画面が表示されないため、アプリ内の「エクスポート」機能を使ってデータを別環境（Vercelなど）に移行することが困難な状況です。

大切なデータを失わないための「復旧・救出策」として、以下のいずれかの方法をご案内します。

## 方法1: Google Cloudの課金設定を再開する（最も確実・簡単）
サーバーを一次的に復旧させ、アプリ画面を開けるようにする方法です。
1. [Google Cloud Consoleの「お支払い」ページ](https://console.cloud.google.com/billing) にアクセスします。
2. 対象の課金アカウント（Account ID: `01A7D1-xxxxx`）のステータスを確認し、クレジットカード情報の更新などを行ってアカウントを「有効（Open）」に戻します。
3. 数分〜数十分待つと自動的にサーバーの凍結が解除され、アプリの画面が開けるようになります。
4. 画面が開けたら、元々備わっている「データエクスポート機能（JSON書き出し）」でデータをダウンロードし、別の無料環境（Vercelなど）へ移行します。

## 方法2: PCブラウザの「開発者ツール」を使ってデータを手動で取り出す
もし該当のURLを開いていたのが **「PCのGoogle Chrome または Edge」** の場合、503エラーが出ている画面でもデータを救出可能です。（※スマホのブラウザの場合はこの操作はできません）
1. PCで当該エラー画面 (`haze-strategy-web.an.r.appspot.com`) を開きます。
2. キーボードの `F12` キー（Macの場合は `Cmd + Option + I`）を押して「開発者ツール」を開きます。
3. 上部タブから **「Application（アプリケーション）」** を選択します（隠れている場合は `>>` アイコンから探します）。
4. 左サイドバーの `Storage` > `Local Storage` > `https://haze-strategy-web.an.r.appspot.com` をクリックします。
5. 右側に表示される `bayesTasks`, `bayesHypotheses` などのキーと値のペアを一つずつコピーし、手元にテキストファイル等で控えます。

---
**今後の対応について**
上記いずれかの方法で手元にデータを救出できましたら、以前ご提案した完全無料のVercelへのデプロイを完了させ、その新しいURL上でデータを復元できるようにサポートいたします。
