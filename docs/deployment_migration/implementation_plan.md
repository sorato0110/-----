# 無料ホスティングへのデプロイ移行計画

## 目的とゴール
App Engine (GAE) の課金エラーによってウェブサイトが閲覧できなくなっている問題を解決するため、完全無料で利用できるホスティングサービスへの移行を行います。

## エラー原因の特定（ブラウザ確認結果）
ブラウザエージェントを使用して `haze-strategy-web.an.r.appspot.com` にアクセスした結果、画面に以下のメッセージが表示されることを確認しました。
> **エラー: サーバーエラー**
> 要求されたサービスはまだ利用できません。30秒後にもう一度お試しください。

**原因:** 
これはステータスコード `503 Service Unavailable` に該当し、Google Cloud アカウントの課金情報が無効になっているため、App Engineのサーバーインスタンスが起動を拒否されていることが直接の原因です。

## 移行先の提案とアーキテクチャ
現在お客様の環境にはNode.js等がインストールされていないため、ターミナルからの複雑なコマンド操作（CLI）を必要としない方法が最適です。
以下の2つの無料ホスティングサービスを提案します。**Vercel（バーセル）** の利用が最も簡単で確実なため推奨いたします。

### 候補1: Vercel（推奨・最も簡単）
Vercelは、GitHubリポジトリと連携するだけで**完全無料かつ全自動**でデプロイが完了する世界中で人気のホスティングサービスです。
- **メリット**:
  - GitHubにコードをPushするだけで自動で最新版がウェブ上に反映されます。
  - カスタムの設定ファイル（`app.yaml`のようなもの）は一切不要です。
  - プライベートリポジトリでも無料でデプロイ可能です。
- **今後の実装ステップ (お客様で行っていただく作業)**:
  1. [Vercel](https://vercel.com/) にアクセスし、GitHubアカウントでログインする。
  2. 「Add New...」>「Project」をクリックし、ご自身のGitHubリポジトリをインポートする。
  3. 設定は初期値のまま「Deploy」ボタンを押すだけで完了です。

### 候補2: GitHub Pages
GitHubが標準で提供している静的サイトホスティング機能です。
- **メリット**: 新たな外部サービスのアカウントを作る必要がありません。
- **デメリット**: GitHubリポジトリが「Private（非公開）」に設定されている場合、無料ユーザーはGitHub Pagesを利用できません（Publicに変更する必要があります。また Firebase 連携など少し設定が必要です）。
