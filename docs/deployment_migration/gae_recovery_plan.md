# App Engine復旧とデータ移行プラン

デバイスのデータ消失を防ぐため、以下の手順で進めます。
作業は大きく3つのステップに分かれます。

## 【Step 1】 サーバーの復旧（お客様の作業）

現在凍結されている App Engine サーバー（元のURL）を一時的に復活させます。これをしないと過去のデータを取り出せません。

1. [Google Cloud コンソール - お支払いの管理](https://console.cloud.google.com/billing) にアクセスします。
2. 『**zhuweikongdong@gmail.com**』のアカウントでログインしていることを確認します。
3. 閉鎖されている課金アカウントを選択し、「**アカウントを再開**」または「**お支払い方法の更新（クレジットカード情報の登録など）**」を完了させてください。
4. 設定が反映されると、数分〜数十分で以前のURL（ `https://haze-strategy-web.an.r.appspot.com` ）にアクセスできるようになり、これまでのデータが表示されます。

=> **この作業が完了し、元のURLでデータが表示されるようになったら「完了した」とお知らせください。**

---

## 【Step 2】 データの救出と本番移行準備（AIの作業）

元のURLが復旧したことを確認させていただいた後、私（AI）が以下の作業を自動で進めます。

1. **データ退避機能の実装**: 現在開発中の「アカウント機能（Firebase）」をオンにし、デバイス内（ローカルストレージ）のデータをクラウドデータベース上に安全にコピーする仕組みを現在のコードに追加します。
2. **アップデートのデプロイ**: 上記の機能を一度、**現在の App Engine サーバー**に対してデプロイします（これまで通りのデプロイ作業を私が実行します）。

---

## 【Step 3】 データのクラウド同期と無料環境への移行（お客様とAIの共同作業）

1. **データのクラウドへのアップロード（お客様の作業）**:
   - 復旧した元のURLにお手元のデバイス（過去のデータが入っている端末）からアクセスしていただきます。
   - 画面に追加された「ログイン」ボタンからアカウント認証を行い、これまでのデータをクラウドに同期させます。
2. **完全無料環境への移行（AIの作業）**:
   - データがクラウドに安全に保管されたことを確認後、App Engine（有料・課金必須）を終了します。
   - その後、完全無料のVercelなどを利用して新環境へデプロイし、以後はそちらの新しいURL（例: `https://haze-strategy.vercel.app`）をご利用いただけるようにすべて設定いたします。
