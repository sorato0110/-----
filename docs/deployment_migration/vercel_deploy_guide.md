# Vercelを利用したデプロイ手順書

App Engineで発生している課金エラーを回避し、完全無料でウェブサイトを公開するため、Vercel（バーセル）を利用したデプロイ手順をご案内します。

お客様の環境はすでにGitHubリポジトリ（`sorato0110/-----`）にコードがプッシュされている状態ですので、以下の手順をブラウザ上で行うだけで**数分でデプロイが完了します**。

## 手順

### 1. Vercelへのサインアップ / ログイン
1. ブラウザで [Vercel公式サイト (https://vercel.com/)](https://vercel.com/) にアクセスします。
2. 画面右上の **「Sign Up」**（または **「Log In」**）をクリックします。
3. アカウント作成方法として **「Continue with GitHub」** を選択し、ご自身のGitHubアカウント（`sorato0110`）で紐付け認証を行います。

### 2. GitHubからのリポジトリインポート
1. ログイン後のVercelダッシュボード画面右上にある **「Add New...」** ボタンをクリックし、**「Project」** を選択します。
2. 「Import Git Repository」という画面が表示されます。
   - `sorato0110` のアカウント直下にある本プロジェクトのリポジトリ（`-----`）が表示されるはずです。
   - その横にある **「Import」** ボタンをクリックします。
     ※もしリポジトリが表示されない場合は、「Adjust GitHub App Permissions →」からVercelに対してインポートしたいリポジトリへのアクセス権限を許可してください。

### 3. デプロイの実行
1. 「Configure Project」という設定画面に移動します。
2. フレームワーク設定などは **すべてデフォルトのままで問題ありません**。（Node.jsなどのビルドは不要な静的サイト構成のため）
3. 画面右下の **「Deploy」** ボタンをクリックします。

### 4. 完了とURLの発行
1. 約数十秒待つと、画面全体に花吹雪（コンフェッティ）が舞い、"Congratulations!" と表示されれば**デプロイ成功**です。
2. 表示された **「Continue to Dashboard」** をクリックします。
3. ダッシュボードの左上に「Domains」として表示されているURL（例: `haze-strategy-xxxxx.vercel.app`）をクリックすると、公開された無料のウェブサイトにアクセスできます。

> [!TIP]
> 今後、HTMLやJSファイルなどのコードを修正して **GitHubの `main` ブランチにプッシュ（Sync）するだけ**で、Vercelがそれを検知して**自動で最新バージョンに再デプロイ**してくれます。手動でのデプロイコマンドはもう必要ありません。
