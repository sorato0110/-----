# Firebase (Authentication + Firestore) 設定手順書

## 1. 料金について
結論から申し上げますと、**完全無料（Sparkプラン）**でご利用いただけます。
個人開発や小規模アプリケーションであれば、Firebaseの無料枠は非常に大きく、無料の範囲を超えることはまずありません。クレジットカードの登録も不要で開始できます。
- **Authentication (認証)**: 月間50,000アクティブユーザーまで無料
- **Cloud Firestore (データベース)**: 1GBの保存容量、月間50,000回の読み込み、20,000回の書き込みが毎日無料

## 3. 具体的な設定ステップ

### ステップ 1: Firebase プロジェクトの作成
1. [Firebase Console](https://console.firebase.google.com/) にアクセスし、Googleアカウントでログインします。
2. **「プロジェクトを追加」** または **「プロジェクトを作成」** をクリックします。
3. **プロジェクト名** を入力します（例: `haze-strategy-app`）。
4. 「Google アナリティクスを有効にする」は今回はオフ（チェックを外す）にして、「プロジェクトを作成」をクリックします。
5. 「プロジェクトの準備ができました」と表示されたら「続行」をクリックします。

### ステップ 2: Authentication（Googleログイン）の有効化
1. 左側のメニューから **「構築」** > **「Authentication」** をクリックします。
2. **「始める」** をクリックします。
3. 「Sign-in method（ログイン方法）」のタブから **「Google」** をクリックします。
4. 右上のトグルスイッチをクリックして **「有効にする」** にします。
5. 「プロジェクトのサポートメール」でご自身のメールアドレスを選択し、下部の **「保存」** をクリックします。

### ステップ 3: Cloud Firestore（データベース）の作成
1. 左側のメニューから **「構築」** > **「Firestore Database」** をクリックします。
2. **「データベースの作成」** をクリックします。
3. **「テストモードで開始する」** を選択し、「次へ」をクリックします。
   *(※テストモードは誰でも開発用に読み書きできるモードです。実装後にセキュリティルールを安全なものに書き換えるサポートもいたします)*
4. ロケーション（データの保存場所）は `asia-northeast1` (東京) または `asia-northeast2` (大阪) を選択し、「作成」をクリックします。

### ステップ 4: Webアプリの登録と設定コードの取得
1. Firebase Consoleの左上の「プロジェクトの概要（🏠アイコンの横）」をクリックしてホーム画面に戻ります。
2. 画面中央のアイコンから **「</> (Web)」** をクリックします。
3. **アプリのニックネーム** を入力します（例: `haze-web`）。
4. 「Firebase Hosting も設定しますか？」のチェックは **外したまま** にして、「アプリを登録」をクリックします。
5. **「Firebase SDK の追加」** という画面が表示されます。
   コードの中に以下のような **`firebaseConfig`** の部分があります。これをコピーしてください。

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyBxxxxxxx...",
  authDomain: "xxxx.firebaseapp.com",
  projectId: "xxxx-project",
  storageBucket: "xxxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdefg"
};
```

6. コピーできたら、「コンソールに進む」をクリックして終了です。
